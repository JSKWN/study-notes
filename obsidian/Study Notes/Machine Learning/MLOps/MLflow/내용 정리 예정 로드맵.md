## 3. 모델 산출물 관리

| 순서  | 제목                                  | 주요 내용                                                                                                          |
| --- | ----------------------------------- | -------------------------------------------------------------------------------------------------------------- |
| 6   | MLflow 산출물(Artifacts) 폴더            | Artifacts(산출물)의 정의와 mlruns/ 폴더 내에서 실험(Experiment) 및 실행(Run) 단위로 모델 파일, 환경 설정 등이 저장되는 계층적 구조 이해.                |
| 7   | MLflow Artifacts 활용, 모델 복원 및 예측 수행  | 저장된 아티팩트(모델 파일, 환경 설정)를 활용하여 이전 실험을 그대로 복원하고 `mlflow.pyfunc.load_model()`을 통해 모델을 로드하여 예측을 수행하는 방법.            |
| 8   | MLflow의 Signature와 Input Example 이해 | 모델 저장 시 입력 및 출력 데이터의 형식(type, shape)을 정의하는 Signature와 실제 입력 샘플을 제공하는 Input Example의 역할 및 안정성 보장 기능.            |
| 번외  | MLflow Model API                    | 프레임워크에 독립적으로 모델을 표준화하여 저장(`save_model`), 서버에 로깅(`log_model`), 불러오기(`load_model`) 위한 통합 인터페이스(`Model API`) 사용법. |

## 4. 모델 배포 및 실험 자동화

| 순서  | 제목                                              | 주요 내용                                                                                                                              |
| --- | ----------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------- |
|     | 모델 등록과 버전 관리, MLflow로 간편하게                      | Registry를 통해 모델을 등록하고 버전을 관리하며, `@staging`, `@production`, `@archived`와 같은 별칭(Alias)**을 사용하여 운영 단계를 구분하는 방법.                       |
|     | MLflowClient 완전 정리                              | MLflow의 Tracking 및 Registry 기능을 코드로 직접 제어하기 위한 저수준 API인 MlflowClient의 역할과 `log_param`, `transition_model_version_stage` 등의 주요 메서드. |
|     | MLflowClient — 실험(Experiment)과 실행(Run) 관리 자동화하기 | MlflowClient를 사용하여 SQL과 유사한 구문으로 실험을 검색하고, 실험/실행을 삭제 및 복원하는 등 대규모 환경을 자동 관리하는 방법.                                                  |
|     | Model Serving                                   | 등록된 Staging/Production 모델을 로드하고, `mlflow models serve` 명령어를 사용하여 모델 예측을 REST API 형태로 쉽게 배포하는 방법.                                   |

| 순서  | 제목                                        | 주요 내용                                                                                        |
| --- | ----------------------------------------- | -------------------------------------------------------------------------------------------- |
|     | MLflow Projects — “기록에 신뢰를 더하는 환경 통일 시스템” | Tracking만으로는 보장되지 않는 실행 환경의 통일성을 `MLproject` 파일과 의존성 파일로 보장하여 재현성을 높이는 시스템 이해.               |
|     | MLflow Projects — 폴더 구조부터 실행까지 한 번에 이해하기  | `MLproject` 파일의 구조(name, entry_points, parameters) 및 `mlflow.projects.run()`을 사용한 실행 자동화 과정. |
|     | MLflow Model Customization                | `mlflow.pyfunc.PythonModel`을 상속받아 전처리/후처리/앙상블 로직을 하나의 모델 객체에 통합하여 유연한 확장성을 확보하는 방법.          |
|     | MLflow Custom Flavor 만들기                  | MLflow가 기본적으로 지원하지 않는 모델을 표준 규격(PyFunc)으로 통합하여 저장, 로드, 서빙할 수 있도록 사용자 정의 Flavor를 정의하는 방법.     |
|     | MLflow Custom Flavor — 코드 흐름과 동작 순서 분석    | Custom Flavor의 내부 동작 순서(저장, `_load_pyfunc()`, 로드, 예측 위임)를 분석하여 MLflow가 낯선 객체를 다루는 원리 이해.     |
